In your repo tomorrow morning

Six things that weren't there last night.

Every Night Shift run leaves a labelled PR per task — night-shift — so gh pr list --label night-shift is the audit trail. The bundle is obvious from the PR title (night-shift/plan:, night-shift/bug:, etc.).

Implemented plans

Picks the next phase from a docs/*-PLAN.md file and opens a PR for it.

Updated docs

Changelog entries, user guide pages, decision records — kept in sync with what landed.

New test coverage

Fills coverage gaps following the patterns your existing tests already use.

Accessibility fixed

WCAG AA violations on key pages — caught and patched before users hit them.

Translated UI

Hardcoded strings extracted into your i18n system so locales stay complete.

Audit PRs

Security, bugs, SEO, and performance — one PR per area, scoped to what's worth fixing.

The bundles

Five shifts. One source of truth.

manifest.yml is the single source of truth for what tasks exist, what they do, and what bundle they belong to. Edit one file to add, rename, or reorder.

Find what's wrong

triage-ci

Explain every red check

Comments on each failed or cancelled check on Night Shift PRs. Re-runs cancellations and clearly-unrelated flakes; leaves real failures for a human.

  • triage-ci-failures

Two ways to run it

Pick the backend that matches how you work.

Same tasks, same task picker, same PR output. Choose during setup — switch later by re-running /night-shift.

Claude Routine

Runs on your Claude account via Claude Code routines. Perfect for personal projects and small teams.

  • Included with your Claude subscription — no API key needed
  • Five routines: build, code-fixes, audits, docs, triage
  • Default schedule fires on weeknights only — quiet weekends
  • Atlassian Rovo for optional Jira pickup

GitHub Actions

Runs on GitHub-hosted runners via a reusable workflow. Built for orgs and shared repos.

  • Scales across an org without consuming Claude account quota
  • One ANTHROPIC_API_KEY org secret — set once
  • Caller workflow installed per repo by the setup command
  • Same task picker, same PR labels, same audit trail
